# Content Rules: Authenticity & Quality Standards

## Authenticity Framework

### Core Principle

Every claim on LinkedIn must be:
1. **DOCUMENTED** - Exists in CONTEXT files (Experience Details.md, Technical Skills.md)
2. **METRIC-BACKED** - Quantified where possible
3. **NOT OVERSOLD** - Doesn't violate Honest Gaps.md constraints
4. **DEFENSIBLE** - Could explain in an interview without hesitation

### Master Context Cross-Check

| Check | Action |
|-------|--------|
| **Underselling** | Flag achievements in CONTEXT not on profile |
| **Overselling** | Flag claims with no backing in CONTEXT |
| **Skill Inflation** | Compare claimed proficiency to actual documented proficiency |
| **Title Inflation** | Ensure job titles match actual titles held |

### Validation Before Output

Before any claim goes on profile:

```
1. Search Experience Details.md - Does this achievement exist?
2. Check Honest Gaps.md - Is this area flagged as "don't oversell"?
3. Verify metrics - Can you explain this number in an interview?
4. Apply skeptic test - Would a hiring manager say "prove it"?
```

Claims failing ANY check are EXCLUDED from profile. NO EXCEPTIONS.

## Voice & Tone Rules

### First-Person Voice (MANDATORY)

LinkedIn is personal. Always use first-person voice.

| Wrong (Resume Style) | Correct (LinkedIn Style) |
|----------------------|--------------------------|
| "Managed a team of 5" | "I led a team of 5" |
| "Responsible for..." | "I drive..." |
| "Results-driven professional" | "I focus on results because..." |
| Third-person bio | First-person narrative |

### Tone Guidelines

| Audience | Tone | Example |
|----------|------|---------|
| Recruiters | Professional, keyword-rich | "Senior Business Analyst with expertise in..." |
| Hiring Managers | Results-focused, specific | "I reduced process time by 40% by..." |
| Peers/Network | Conversational, authentic | "I'm passionate about solving complex operational problems..." |

### Words to Avoid

**Generic Phrases (NO VALUE):**
- "Passionate" (unless followed by specific evidence)
- "Team player"
- "Results-driven"
- "Hard worker"
- "Go-getter"
- "Think outside the box"
- "Synergy"
- "Leverage"

**Replacement Strategy:**
| Instead of | Use |
|------------|-----|
| "Passionate about X" | "I've spent 3 years building X because..." |
| "Results-driven" | "I increased revenue 40% by..." |
| "Team player" | "I collaborate with engineering and sales to..." |
| "Hard worker" | [Show don't tell - describe what you accomplished] |

## Keyword Integration Rules

### Natural Density Targets

| Section | Target Density | Method |
|---------|---------------|--------|
| Headline | 2-3 keywords | Every word counts, no filler |
| About | 5-10% of content | Naturally woven into narrative |
| Experience | 3-5 per role | In bullets, action verbs |
| Skills | Direct matches | From job posting analysis |

### Keyword Placement Strategy

**Priority Order:**
1. First 60 characters of headline (search-indexed)
2. First sentence of About section
3. First bullet of each Experience entry
4. Skills section (exact matches)

### Avoid Keyword Stuffing

**Bad Example:**
```
"AI specialist, AI expert, AI implementation, AI automation,
AI consultant with AI experience in AI projects"
```

**Good Example:**
```
"I help companies implement AI automation that actually works.
At [Previous Company], I built systems that reduced manual work by 70%."
```

## Quality Standards

### Headline Quality Check

| Check | Pass Criteria |
|-------|---------------|
| Length | ≤ 220 characters |
| Keywords | Primary keyword in first 60 chars |
| Value | Clear value proposition visible |
| Differentiation | Not generic ("Business Analyst") |
| No Errors | No typos, proper capitalization |

### About Section Quality Check

| Check | Pass Criteria |
|-------|---------------|
| Length | 1800-2200 chars (max 2600) |
| Hook | First 300 chars create curiosity |
| Structure | Hook → Why → Expertise → Proof → CTA |
| Voice | First-person throughout |
| Metrics | At least 3 quantified achievements |
| CTA | Clear next step (contact info or "Let's connect") |

### Experience Quality Check

| Check | Pass Criteria |
|-------|---------------|
| Bullets | 3-5 per role |
| Metrics | At least 1 quantified bullet per role |
| Voice | First-person action verbs |
| Impact | Focus on outcomes, not duties |
| Verification | All claims in Experience Details.md |

## Ruthless Feedback Calibration

Use these to calibrate internal quality review:

| Issue | Ruthless Feedback |
|-------|-------------------|
| Generic headline | "You sound like everyone else. 'Passionate Problem Solver' says nothing." |
| Buzzword salad | "'Dynamic, results-driven professional' - what do you actually DO?" |
| No value prop | "Why should I click your profile? You haven't given me a reason." |
| Vague About | "'I love solving complex problems' - what problems? For who? What result?" |
| Missing metrics | "'Improved processes' - by how much? This means nothing without numbers." |
| First-person overuse | "Every sentence starts with 'I'. It's exhausting. Vary your structure." |
| Overclaiming | "You managed '3 junior analysts' - that's not 'team leadership'. Be honest." |

## Error Recovery

| Error | Recovery Action |
|-------|-----------------|
| Claim not in CONTEXT | Remove claim, note in output what was removed |
| Honest Gaps violation | Flag in output, suggest alternative framing |
| Character limit exceeded | Trim per priority: Authenticity > Limits > Keywords |
| Keyword density too high | Rewrite for natural language |
| Generic language detected | Replace with specific examples |
| Voice inconsistency | Rewrite to first-person |
